# Contributing

Thanks for helping improve BarqDB.

## Before You Start

- Open an issue for large changes.
- Keep pull requests small when possible.
- Include tests or notes on manual testing.
- Update docs when behavior changes.

## Pull Requests

Good pull requests include:

- A clear summary
- The reason for the change
- Test results
- Any risk or follow-up work

## Local Setup

Each repository has its own build steps. Start with that repository's README.
